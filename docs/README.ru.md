# Xian-MAGE — Ассистент реального времени на базе компьютерного зрения и обработки языка для игровых сред 🧙‍♂️🎮🤖

Xian-MAGE — это работающий в реальном времени и постоянно присутствующий на рабочем столе игровой HUD и ассистент для Linux (Wayland). Работая на базе бэкенда **[Lemonade Server](https://lemonade-server.ai/)**, он выводит распознавание текста (OCR) в реальном времени, перевод, визуализаторы кликов с визуальной привязкой и интерактивный чат непосредственно поверх активных игровых сред.

Поскольку процесс вывода модели оркестрируется через Lemonade, **MAGE поддерживает выполнение с ускорением Vulkan, плавно работая на графических процессорах AMD Radeon™ и других ускорителях.**

## 🌟 Основные функции (Полностью работоспособны)

- **Аппаратное ускорение Vulkan / AMD GPU**: Выполнение визуально-языковых моделей с низкой задержкой благодаря аппаратному ускорению на стороне GPU.
- **Прозрачный оверлей с пропусканием кликов**: Прозрачные окна-оверлеи на базе PyQt6, которые отображают переведенный текст прямо поверх игровых HUD и диалогов, оставаясь при этом полностью прозрачными для кликов мыши.
- **Глобальные горячие клавиши Wayland и командный OSD**: Бесшовный вызов перевода, настроек OSD и боковых панелей с помощью настраиваемых системных горячих клавиш.
- **Режим диалога (автопрохождение визуальных новелл / сюжетных RPG)**: Зафиксируйтесь на области экрана, автоматически переводите и продвигайте/обновляйте переводы по клику мыши.
- **Подсветка целей с визуальной привязкой**: Спросите ассистента *"куда мне кликнуть?"* или *"где выход?"*, и наблюдайте, как он подсвечивает точные физические координаты на вашем экране.
- **Кинематографический режим (контекстный перевод озвучки)**: Бесшовно объединяет анализ зрения при захвате экрана с переводом аудиовоспроизведения.
- **Локальный словарь CC-CEDICT**: Наведите курсор на любой пузырь с переводом и нажмите `Alt` для локального, потокобезопасного разбора китайских иероглифов, пиньиня и определений.

## 🛠️ Начало работы (Клиент MAGE)

### Требования

- **Linux с Wayland** (X11 также поддерживается; для глобальных привязок клавиш требуется ввод через `evdev`)
- **Права пользователя**: Ваш пользователь должен быть в группе `input` для перехвата глобальных горячих клавиш (`sudo usermod -aG input $USER` и выполните выход/вход в систему)
- **Lemonade Server**: Запущенный экземпляр Lemonade Server (по умолчанию доступен по адресу `http://localhost:13305`)

### Быстрая настройка (Linux)

Клонируйте репозиторий и запустите скрипт инициализации — он установит [`uv`](https://docs.astral.sh/uv/), синхронизирует все зависимости и автоматически запустит MAGE:

```bash
git clone https://github.com/clemperorpenguin/xian-vl.git
cd xian-vl
./mage.sh
```

Чтобы добавить MAGE в меню приложений рабочего стола:

```bash
./mage.sh --install
```

Чтобы добавить MAGE в меню **и** автоматически установить системные зависимости, собрать встраиваемый сервер Lemonade из исходного кода и загрузить модель визуального языка по умолчанию:

```bash
./mage.sh --install --build
```

Чтобы удалить запись рабочего стола и иконку:

```bash
./mage.sh --uninstall
```

### Готовые сборки (Windows и macOS)

Если вы используете Windows или macOS, или предпочитаете не собирать проект из исходного кода, загрузите готовые пакеты со страницы [GitHub Releases](https://github.com/clemperorpenguin/xian-vl/releases).

Сборки доступны в двух вариантах:
- **Lite**: Облегченная автономная версия. Требует подключения к внешнему запущенному серверу Lemonade Server.
- **Full**: Поставляется со встроенным сервером `lemond`, который автоматически запускается и останавливается вместе с приложением.

#### Windows
1. Загрузите `mage-client-Windows-x86-64-lite.zip` или `mage-client-Windows-x86-64-full.zip`.
2. Распакуйте архив.
3. Дважды щелкните `mage-client.exe` для запуска.

#### macOS
1. Загрузите `mage-client-MacOS-ARM64-lite.dmg` или `mage-client-MacOS-ARM64-full.dmg` (или эквиваленты в формате ZIP).
2. Дважды щелкните DMG и перетащите `mage-client.app` в папку **Программы** (Applications).
3. Откройте и запустите приложение.
   > [!NOTE]
   > Поскольку приложение не подписано и не прошло нотариальное заверение Apple, Gatekeeper заблокирует его с предупреждением о том, что приложение "повреждено и не может быть открыто". Вы можете легко исправить это, выполнив следующую команду в терминале:
   > ```bash
   > xattr -cr /Applications/mage-client.app
   > ```

### Ручная настройка (Все платформы)

Если вы предпочитаете управлять окружением самостоятельно:

```bash
git clone https://github.com/clemperorpenguin/xian-vl.git
cd xian-vl
uv sync --all-packages
uv run --package mage-client mage
```

### Управление
- **Открыть меню действий / OSD** — Двойное нажатие `Shift` (Горячая клавиша-лидер по умолчанию)
- **Захват экрана** — `C` в меню действий (выберите область экрана, затем Перевод / Диалог / Чат)
- **Переключить боковую панель чата** — `A` в меню действий
- **Перевести для чата (Ввод)** — `T` в меню действий
- **Панель настроек** — `S` в меню действий

## 📁 Архитектура и смежные проекты

Монорепозиторий содержит основной готовый к продакшену клиент MAGE, а также экспериментальные сопутствующие проекты:

```
├── apps/
│   ├── mage-client/      # 🧙‍♂️ Основное проверенное приложение игрового HUD на базе PyQt6
│   ├── nate/             # 📱 Android-компаньон: OCR-ридер и словарь (экспериментальное)
│   ├── masha-extension/  # 🌐 Браузерное расширение для перевода выделенного текста (экспериментальное)
│   ├── lore-client/      # 📜 CLI-инструмент для создания базы знаний RAG (экспериментальное)
│   └── luduan-client/    # 🦤 CLI для перевода EPUB и создания аудиокниг (экспериментальное)
└── packages/
    ├── xian-vl/          # ⚙️ Базовый движок оркестрации LLM/ASR и менеджеры контекста
    └── shared-types/     # 📦 Канонические модели, константы и общие типы
```

> [!WARNING]
> **Ограничение ASR / Аудио**: Отправка загрузок потокового аудио в Lemonade в настоящее время не работает из-за ограничений бэкенда на стороне сервера. Следовательно, функции перевода живой речи (такие как **Режим рейда**) отключены. Все функции визуального OCR, перевода текста, словаря и визуальной привязки в чате полностью работоспособны.
